# 情报系统优化开发计划

## 一、目标
构建一个智能化的ETF情报管理系统，实现：
1. 统一情报库（不重复存储）
2. LLM自动关联ETF
3. 自动归档和压缩
4. 按ETF展示专属页面

---

## 二、系统架构

```
┌─────────────────────────────────────────────────────┐
│                    情报输入层                        │
│  用户手动输入 / 新闻API / 爬虫                    │
└──────────────────────┬──────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────┐
│                  LLM处理层                         │
│  1. 关键词提取                                   │
│  2. 情感判断（利好/利空）                        │
│  3. 自动关联ETF + 置信度                         │
│  4. 自动摘要                                     │
└──────────────────────┬──────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────┐
│                   存储层                           │
│  ┌─────────────┐  ┌─────────────┐               │
│  │ 活跃情报   │  │ 历史归档   │               │
│  │ (6个月内)  │  │ (压缩存储) │               │
│  └─────────────┘  └─────────────┘               │
│  ┌─────────────┐  ┌─────────────┐               │
│  │ 向量库     │  │ 关联表     │               │
│  │ (语义搜索) │  │ (情报-ETF) │               │
│  └─────────────┘  └─────────────┘               │
└──────────────────────┬──────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────┐
│                   展示层                           │
│  每只ETF专属页：情报列表 + 持仓 + 走势           │
└─────────────────────────────────────────────────────┘
```

## 三、开发阶段

### Phase 1: 数据结构优化（优先级：高）
**时间：1-2天**

| 任务 | 说明 |
|------|------|
| 1.1 新增字段 | 添加 is_archived, summary 字段 |
| 1.2 创建关联表 | intelligence_stocks (情报-ETF多对多) |
| 1.3 创建ETF配置表 | etf_keywords (ETF关键词配置) |
| 1.4 迁移数据 | 把现有情报导入新结构 |

**数据库改动：**
```sql
-- 新增字段
ALTER TABLE intelligence ADD COLUMN is_archived BOOLEAN DEFAULT 0;
ALTER TABLE intelligence ADD COLUMN summary TEXT;

-- 关联表
CREATE TABLE intelligence_stocks (
    intelligence_id INTEGER,
    symbol TEXT,
    confidence REAL,
    PRIMARY KEY (intelligence_id, symbol)
);

-- ETF配置表
CREATE TABLE etf_keywords (
    symbol TEXT PRIMARY KEY,
    name TEXT,
    keywords TEXT,  -- JSON数组
    tags TEXT,     -- JSON数组
    updated_at TIMESTAMP
);
```

### Phase 2: LLM智能关联（优先级：高）
**时间：2-3天**

| 任务 | 说明 |
|------|------|
| 2.1 LLM关联接口 | 调用DeepSeek分析情报，自动关联ETF |
| 2.2 关键词配置 | 可配置的ETF关键词 |
| 2.3 置信度阈值 | 默认>80%自动关联 |

**LLM提示词：**
```
你是一个金融情报分析助手。请分析以下情报，输出关联的ETF代码。

用户关注的ETF：
- 588200: 科创芯片ETF，关键词：芯片、半导体、中芯国际...
- 588710: 科创半导体设备ETF，关键词：半导体设备、光刻机...
- 588750: 科创芯片ETF汇添富

情报内容：{content}

请输出JSON格式：
{{"symbol": "588200", "confidence": 0.95}, ...}}
```
