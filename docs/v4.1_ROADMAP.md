## 🎯 v4.1 三大核心模块开发计划

### 模块一: 🧠 投研情报系统涅槃重构 (Intelligence Hub 2.0)

**目标**: 解决情报过多导致token超限、检索效率低的问题

#### Phase 1: 数据库结构优化 (Week 1)
- [ ] **1.1 新增冷热分离字段**
  ```sql
  ALTER TABLE intelligence ADD COLUMN is_archived BOOLEAN DEFAULT 0;
  ALTER TABLE intelligence ADD COLUMN summary TEXT;
  ALTER TABLE intelligence ADD COLUMN confidence REAL;
  ```
- [ ] **1.2 创建情报-ETF关联表**
  ```sql
  CREATE TABLE intelligence_stocks (
      intelligence_id INTEGER,
      symbol TEXT,
      confidence REAL,
      PRIMARY KEY (intelligence_id, symbol)
  );
  ```
- [ ] **1.3 ETF关键词配置表**
  ```sql
  CREATE TABLE etf_keywords (
      symbol TEXT PRIMARY KEY,
      name TEXT,
      keywords TEXT,  -- JSON
      tags TEXT,      -- JSON
      updated_at TIMESTAMP
  );
  ```

#### Phase 2: LLM智能关联 (Week 1-2)
- [ ] **2.1 DeepSeek智能分析接口**
  - 输入: 情报内容 + 关注ETF列表
  - 输出: 关联ETF代码 + 置信度 + 利好/利空判断
- [ ] **2.2 情报入库流程改造**
  - 新情报 → LLM分析 → 自动关联 → 推送通知
- [ ] **2.3 置信度阈值配置** (默认80%)

#### Phase 3: 自动归档系统 (Week 2)
- [ ] **3.1 定时归档脚本** (`scripts/intel_archive.py`)
  - 每月1日执行
  - 6个月前情报自动归档
- [ ] **3.2 LLM压缩摘要**
  - 500字 → 100字核心信息
- [ ] **3.3 检索优化**
  - 只查活跃情报 (is_archived=0)
  - 按优先级 + 时间排序

#### Phase 4: ETF专属页面 (Week 3)
- [ ] **4.1 情报时间轴组件**
  - 今日情报 / 本周情报 / 历史高价值情报
- [ ] **4.2 持仓股票情报联动**
  - 展示ETF持仓股相关情报
- [ ] **4.3 情绪标签展示**
  - 利好/利空视觉区分

### 模块三: ⚔️ 双子星智能阵列演进 (Dual-Expert Architecture)

**目标**: 风控拦截系统化、AI接口统一化

#### Phase 1: 红蓝决策矩阵上链 (Week 2)
- [ ] **1.1 风险评分编码化**
  - Gemini 风险评分 >7分: 强制预警
  - 禁用实盘一键买入按钮
- [ ] **1.2 决策矩阵实现**
  ```python
  # 在capability_platform/risk/dual_expert_decision.py
  class DualExpertDecision:
      def evaluate(self, blue_advice, red_score):
          if red_score > 7:
              return Decision.REJECT
          elif red_score > 4:
              return Decision.CAUTION
          else:
              return Decision.APPROVE
  ```

#### Phase 2: AI接口统一 (Week 3)
- [ ] **2.1 统一API密钥读取链路**
  - Web界面 & 命令行使用同一配置

- [ ] **2.2 批量风控分析脚本**
  - 支持命令行批量红蓝分析
  - 输出结构化JSON报告
